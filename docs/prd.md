# 需求文档

## 1. 应用概述

### 1.1 应用名称
DeepHarness桌面程序

### 1.2 应用描述
这是一个Web应用，模拟DeepHarness桌面程序的界面和交互体验。用户可通过该应用选择编码智能体，与AI进行编码对话，管理文件、任务和历史会话，并配置智能体、技能、提示词等设置。

## 2. 用户与使用场景

### 2.1 目标用户
需要使用AI辅助编码的开发者或技术人员。

### 2.2 核心使用场景
- 用户登录后选择编码智能体（如OpenCode、Claude Code等）
- 在智能会话窗口与AI进行编码对话
- 查看和管理文件列表、历史会话、任务列表、修改的文件列表
- 配置智能体、模型、主题、技能、提示词、设计规范等设置

## 3. 页面结构与功能说明

### 3.1 页面结构

```
├── 登录页面
├── 智能体选择页面
├── 主界面（智能会话页面）
│   ├── 左侧栏：文件列表 + 历史会话列表
│   ├── 中间区域：智能会话窗口
│   ├── 右侧栏：任务列表 + 修改的文件列表
│   └── 顶部设置按钮（右上角）
└── 设置面板
    ├── 基础设置标签页
    ├── 技能标签页
    ├── 提示词标签页
    └── 设计规范标签页
```

### 3.2 登录页面

#### 3.2.1 功能说明
- 用户输入用户名和密码完成登录
- 用户输入用户名和密码完成注册

### 3.3 智能体选择页面

#### 3.3.1 功能说明
- 展示可选的编码智能体列表，包括OpenCode、Claude Code等选项
- 用户选择一个编码智能体后进入主界面

### 3.4 主界面（智能会话页面）

#### 3.4.1 整体布局
- 左中右三栏布局
- 顶部右上角显示设置按钮

#### 3.4.2 左侧栏
- 上方展示文件列表
- 下方展示历史会话列表

#### 3.4.3 中间区域
- 智能会话窗口，用户在此与AI进行编码对话
- 用户输入消息，AI返回回复

#### 3.4.4 右侧栏
- 上方展示任务列表
- 下方展示修改的文件列表

#### 3.4.5 顶部设置按钮
- 点击后打开设置面板

### 3.5 设置面板

#### 3.5.1 基础设置标签页
- 配置当前编码智能体（OpenCode、Claude Code等）
- 选择模型
- 切换主题

#### 3.5.2 技能标签页
- 技能槽配置
- 包含「需求设计」下拉技能选择
- 包含「开发编码」下拉技能选择

#### 3.5.3 提示词标签页
- 卡片式提示词列表
- 搜索功能：用户输入关键词搜索提示词
- 标签筛选：包含产品设计、编码等标签，用户选择标签筛选提示词

#### 3.5.4 设计规范标签页
- 显示全局的claude.md内容
- 显示项目的claude.md内容

## 4. 业务规则与逻辑

### 4.1 登录与注册
- 用户首次使用需完成注册
- 已注册用户可直接登录
- 登录成功后进入智能体选择页面

### 4.2 智能体选择
- 用户必须选择一个编码智能体后才能进入主界面
- 选择的智能体将作为当前会话的AI对话对象

### 4.3 智能会话
- 用户在中间区域输入消息，AI返回回复
- 会话记录保存至历史会话列表

### 4.4 设置配置
- 用户在设置面板中修改配置后，配置立即生效
- 切换编码智能体或模型后，后续会话使用新配置

## 5. 异常与边界情况

| 场景 | 处理方式 |
|------|----------|
| 用户未登录直接访问主界面 | 跳转至登录页面 |
| 用户未选择智能体直接访问主界面 | 跳转至智能体选择页面 |
| 用户输入空消息 | 不发送消息，提示用户输入内容 |
| AI回复失败 | 提示用户「AI回复失败，请重试」 |
| 搜索提示词无结果 | 显示「未找到相关提示词」 |

## 6. 验收标准

1. 用户在登录页面输入用户名和密码完成注册
2. 用户在登录页面输入用户名和密码完成登录
3. 用户在智能体选择页面选择一个编码智能体（如OpenCode）
4. 用户进入主界面，在中间区域智能会话窗口输入编码相关问题
5. AI返回回复，用户查看回复内容
6. 用户点击顶部右上角设置按钮，打开设置面板
7. 用户在基础设置标签页切换主题
8. 用户关闭设置面板，主题切换生效
