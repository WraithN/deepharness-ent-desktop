# 右键菜单作用域导致文件列表误触与会话异常

## 现象

工作区新增右键菜单后，文件列表中的文件项右键也会弹出应用菜单；同时会话主页面在部分右键操作后出现异常崩溃或交互中断。

## 根因

右键菜单启用标记被挂在工作区主内容父容器上，文件列表、右侧栏和会话区都会通过 `closest()` 匹配到该父节点，导致右键事件作用域过大。文件列表未显式阻止右键事件，冒泡后触发了会话页的上下文菜单逻辑。

## 解决方案

将右键菜单触发范围收窄到会话输出列表区域，仅 `data-workspace-context-menu="true"` 的最近区域允许打开菜单；文件列表根节点标记为 `data-workspace-context-menu="false"` 并阻止默认右键菜单，避免误触。验证已通过 `pnpm lint`。
